# ldv-analysis

Refracto-vibrometry analysis of acoustofluidic devices from scanning LDV (laser Doppler vibrometer) TDMS data.

## Setup

```bash
python -m venv .venv
.venv/Scripts/activate   # Windows
pip install -e .
```

## Data

Raw acquisition files (`.tdms` for v1 / `.h5` for v2) are not in the
repository — they sit on OneDrive (or wherever else). Tell the analysis
scripts where to find them via environment variables (or a `.env` file
in the repo root):

| Variable | Purpose | Required? |
|---|---|---|
| `LDV_DATA_ROOT` | Root that contains the experiment subfolders — scripts compose paths as `LDV_DATA_ROOT / "output" / "W21" / <scan_dir>`. | yes |
| `LDV_CACHE_ROOT` | Optional shared root for FFT caches (`<root>/<scan_dir>/fft_cache/`). When unset, each script writes its cache under `experiments/<week>/output/`. | no |
| `MANUSCRIPT_DIR` | Optional output directory for manuscript figures. | no |

A repo-root `.env` is read at import time — same syntax as a shell
file, comments allowed:

```ini
LDV_DATA_ROOT=C:/Users/<you>/OneDrive - Lund University/Data
# Optional: share FFT caches between machines via OneDrive.
LDV_CACHE_ROOT=C:/Users/<you>/OneDrive - Lund University/Data/output/_fft_cache_share
```

Env vars take precedence over `.env`. No code edits required to switch
machines.

### Working remotely / on a machine with limited SSD

The FFT cache (~65 MB for the W21 waveguide-test datasets) is the
output of an expensive step over the raw waveforms (~488 GB) and is all
that the analysis scripts read after the first run. Two ways to skip
shipping the raw data to the remote:

1. **Shared cache** — point `LDV_CACHE_ROOT` at an OneDrive folder on
   both machines. Cache files generated on one are auto-synced to the
   other.
2. **Cache-only mode** — copy the cache once (zip / robocopy / rsync)
   and run the scripts. If the raw `.h5` is missing on the remote,
   `fft_cache.load_or_compute` skips its mtime-freshness check and
   accepts the cache as ground truth; only raises if both the cache
   and the raw file are absent.

### Legacy W09 data layout

The initial W09 scripts still expect the older repo-relative layout
(used by `00_convert_tdms.py`):

```
data/
├── raw/        <- put .tdms and .tdms_index files here
└── converted/  <- generated by 00_convert_tdms.py
```

Newer experiments (W10+) read directly from `LDV_DATA_ROOT` and don't
need a `data/` tree in the repo.

## Project Structure

```
src/ldv_analysis/       Reusable library (io, analysis, config, FFT cache)
experiments/
  2026W09_initial/      Week 9 initial characterization scripts
  2026W10_stepA/        Week 10 Step A frequency sweep / resonance scripts
reports/                Experiment reports (Markdown)
output/                 Generated figures and data (gitignored)
```

## Experiments

### 2026W09_initial

Run in order. Each script reads from `data/converted/` and writes figures to `output/<dataset>/<script_name>/`.

| Script | Description |
|--------|-------------|
| `00_convert_tdms.py` | Convert TDMS to `.npz` for fast loading |
| `01_electrical_input.py` | Drive frequency, voltage, current, and I-V phase |
| `02_velocity_profile.py` | Apparent velocity amplitude and phase along scan line |
| `03_spectrum.py` | Harmonic spectrum (1f, 2f) and pressure estimates |
| `04_2d_map.py` | 2D spatial map from area scan |

### 2026W10_stepA

Burst-mode analysis scripts for Step A frequency sweep experiments.

| Script | Description |
|--------|-------------|
| `single_mode_shape.py` | Single-file mode shape, waveform/spectrum, repeatability |
| `freq_sweep_coarse.py` | Coarse frequency sweep with sinusoidal mode-shape fitting |
| `freq_sweep_fine.py` | Fine frequency sweep around resonance peaks |
| `freq_axial_sweep.py` | Frequency × axial position sweep |
| `pressure_map_2d.py` | 2D pcolormesh maps with boundary detection |
| `pressure_buildup.py` | Time-resolved pressure field evolution |
| `thermal_drift_check.py` | Ch1/Ch4 electrical stability check |
| `transient_ringup_fit.py` | Ring-up/ring-down Q estimation |
